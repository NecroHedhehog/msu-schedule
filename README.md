# MSU Schedule

Сервис расписания для студентов МГУ. Автоматический сбор расписания с сайтов факультетов, Telegram-бот для быстрого доступа, веб-интерфейс.

## Возможности

- Автоматический сбор расписания со всех групп и курсов
- Telegram-бот: выбор группы кнопками, расписание на день/неделю, навигация
- Персональный фильтр предметов по выбору
- Уведомления об изменениях в расписании (WIP)
- Единая БД для бота и веб-интерфейса

## Архитектура

```
┌─────────────┐     ┌──────────┐     ┌──────────────────┐
│  Парсеры    │────▶│  SQLite  │◀────│  Telegram-бот    │
│  (cron)     │     │          │     │  (aiogram)       │
└─────────────┘     │          │     └──────────────────┘
                    │          │
                    │          │◀────┌──────────────────┐
                    └──────────┘     │  Веб-интерфейс   │
                                    │  (FastAPI) [WIP]  │
                                    └──────────────────┘
```

Парсеры запускаются по расписанию, собирают данные и складывают в SQLite. Бот и сайт читают из одной базы. Компоненты независимы — если парсер упал, бот продолжает отдавать последние данные.

## Стек

- **Python 3.11+**
- **BeautifulSoup4** — парсинг HTML
- **aiogram 3** — Telegram Bot API
- **SQLite** (WAL mode) — хранилище
- **systemd** — деплой на VPS

## Быстрый старт

```bash
git clone https://github.com/YOUR_USERNAME/msu-schedule.git
cd msu-schedule
pip install -r requirements.txt
cp .env.example .env
# отредактируй .env — впиши BOT_TOKEN и т.д.
```

Парсер конфигурируется через `parsers/config/` — необходимо указать параметры целевого сайта расписания (см. `parsers/config/example.json`).

```bash
# собрать расписание
python run_parser.py socio

# запустить бота
python run_bot.py
```

## Структура проекта

```
msu-schedule/
├── core/
│   ├── config.py          # конфигурация, загрузка .env
│   └── database.py        # схема БД, запросы, фильтрация
├── parsers/
│   ├── base.py            # базовый класс парсера
│   ├── socio.py           # реализация для социологического ф-та
│   └── config/            # конфиги парсеров (не в репо)
│       └── example.json   # шаблон конфигурации
├── bot/
│   ├── main.py            # хендлеры, навигация, inline-кнопки
│   └── formatting.py      # форматирование сообщений
├── deploy/                # systemd unit-файлы
├── run_parser.py          # точка входа: парсер
├── run_bot.py             # точка входа: бот
├── requirements.txt
└── .env.example
```

## Деплой

Проект рассчитан на self-hosted VPS (1 vCPU, 1 GB RAM достаточно). Подробности в `deploy/`.
