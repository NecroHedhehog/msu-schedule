# MSU Schedule

Сервис расписания для студентов МГУ — автоматический сбор, Telegram-бот, веб-интерфейс.

Рабочий бот: [@msu_schedule_bot](@msu_schedule_test_bot) (заменить на реальную ссылку)

## Что делает

Парсеры ходят на сайты расписания факультетов МГУ, собирают данные по всем группам и курсам, складывают в единую базу. Telegram-бот даёт студентам доступ к расписанию: выбор группы кнопками, расписание на день/неделю с навигацией, персональный фильтр предметов по выбору. Веб-интерфейс — в разработке.

## Архитектура

```
┌─────────────┐     ┌──────────┐     ┌──────────────────┐
│  Парсеры    │────▶│  SQLite  │◀────│  Telegram-бот    │
│  (cron)     │     │          │     │  (aiogram)       │
└─────────────┘     │          │◀────┌──────────────────┐
                    └──────────┘     │  Веб [WIP]       │
                                    └──────────────────┘
```

Компоненты независимы: парсер упал — бот продолжает отдавать последние данные.

## Стек

Python 3.11, BeautifulSoup4, aiogram 3, SQLite (WAL), systemd на VPS.

## Структура

```
core/           — конфиг, БД (схема, запросы, фильтрация)
parsers/        — парсеры факультетов, конфиги в parsers/config/
bot/            — хендлеры, форматирование, inline-кнопки
deploy/         — systemd unit-файлы [WIP]
run_parser.py   — точка входа парсера
run_bot.py      — точка входа бота
```

## Текущий охват

- **Соцфак** (`cacs.socio.msu.ru`) — 46 групп, бакалавриат + магистратура, все направления и курсы
- **ФГУ** (`cacs.spa.msu.ru`) — в планах, структура разобрана

## Заметки

Расписание соцфака отдаётся помесячно, кодировка cp1251. Навигация четырёхуровневая: отделение → направление → год набора → группа. На одну пару может приходиться несколько предметов по выбору — бот позволяет студенту отметить свои и фильтрует остальные.

Система ФГУ/юрфака — другая платформа (div-блоки вместо таблиц, UTF-8, навигация через `/ttablestd/`). Парсер для неё потребует отдельный адаптер, но формат данных совместим.
