# MSU Schedule

Сервис расписания для студентов МГУ — автоматический сбор, Telegram-бот, веб-интерфейс.

Бот: [@msu_schedule_test_bot](https://t.me/msu_schedule_test_bot)

## Что делает

Парсеры собирают расписание с сайтов факультетов МГУ: групповые расписания, персональные расписания студентов, расписания преподавателей. Данные складываются в единую базу. Telegram-бот даёт студентам мгновенный доступ ко всему.

### Возможности бота

- Выбор группы кнопками (направление → курс → группа) или текстом
- Привязка по фамилии — студент находит себя, группа и предметы ставятся автоматически
- Расписание на сегодня / завтра / неделю с преподавателями
- Навигация по неделям (в пределах доступных данных)
- Фильтр предметов по выбору — автоматический при привязке или ручной
- Поиск расписания преподавателя по фамилии
- Корректное время пар, включая МФК-слоты по средам
- В выходные автоматически показывает следующую неделю
- Нормализация ввода (русская/английская раскладка, ввод без префикса)
- Аналитика пользователей и действий (`/stats`)
- Настраиваемая рекламная система через `.env`
- Алерты в Telegram при ошибках парсера

## Стек

Python 3.11, BeautifulSoup4, aiogram 3, SQLite (WAL), systemd на VPS.

## Структура

```
core/
  config.py          — конфигурация из .env, время пар
  database.py        — схема БД, CRUD, фильтрация, аналитика
  db_students.py     — студенты, преподаватели, привязка, автофильтр
  alerts.py          — Telegram-алерты для админа

  base.py            — базовый класс парсера
  socio.py           — соцфак: группы, студенты, преподаватели
  config/            — конфиги парсеров (не в репо)

```

## Парсинг

Три этапа, запускаются последовательно:

1. **Группы** (`python run_parser.py socio`) — ~100 запросов, 2 минуты. Собирает расписания всех групп.
2. **Студенты** (`python run_parser.py students`) — ~3500 запросов, 30–40 минут. Персональные расписания, пофамильные списки, преподаватели из расписаний студентов.
3. **Преподаватели** (`python run_parser.py teachers`) — ~300 запросов, 2 минуты. Дополняет преподавателей через кафедры + заполняет семинары от лекций.

Результат: 46 групп, 1101 студент, 190 преподавателей, 82% покрытие преподавателями.

## Текущий охват

- **Соцфак** (`cacs.socio.msu.ru`) — 46 групп, бакалавриат + магистратура, все направления и курсы
- **ФГУ** (`cacs.spa.msu.ru`) — структура разобрана, парсер в планах

## Схема данных

```
faculties          — факультеты
groups_            — группы (с привязкой к факультету, направлению, курсу)
lessons            — занятия (предмет, время, аудитория, преподаватель)
students           — пофамильные списки (ФИО, группа, site_id)
student_subjects   — предметы конкретного студента (для автофильтра)
subscriptions      — привязка Telegram → группа
user_subjects      — выбранные предметы пользователя (фильтр)
users              — Telegram-профили (аналитика)
activity_log       — действия пользователей (аналитика)
parse_log          — история парсинга (мониторинг)
```

## Технические заметки

Расписание соцфака: cp1251, PHP, навигация четырёхуровневая (отделение → направление → год набора → группа). Персональные расписания студентов — сессионная навигация (selst → pMns отдельными запросами).

По средам пары 4–5 — МФК-слоты с отдельным временем (15:10–16:40, 17:00–18:30), парсер определяет день недели и подставляет корректное время.

Преподаватели: 77% покрытие из студенческих расписаний, +2% через кафедры, +3% дозаполнение семинаров от лекций = 82% итого. Остальные — МФК и физкультура без преподавателя на сайте.

ФГУ/юрфак — другая платформа (div-блоки, UTF-8, `/ttablestd/`). Отдельный адаптер, формат данных совместим.
